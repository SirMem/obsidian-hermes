# Java集合：List、Set、Map 与 ArrayList、LinkedList

> [!abstract] 一句话摘要
> Java 集合的核心不是“都能存数据”，而是“如何组织数据”：List 强调顺序，Set 强调唯一性，Map 强调键值映射；在线性表实现上，ArrayList 更适合大多数读多写少的业务场景，LinkedList 更适合少数对头尾操作敏感的场景。

---

## 一、先抓总区别：List、Set、Map 到底差在哪？

很多初学者会觉得它们只是“不同的容器”，但真正重要的是：

> **它们管理数据的方式不同。**

### 1. List：按顺序保存一批元素

List 的特点是：

- 有顺序
- 有索引
- 允许重复

你可以把它理解成“排队的队伍”或“数组的增强版”。

适合场景：

- 聊天记录
- 商品浏览历史
- 按录入顺序保存的数据

例如：

```java
[张三, 李四, 张三]
```

这里重复是允许的，顺序也有意义。

### 2. Set：强调不重复

Set 的特点是：

- 不允许重复元素
- 通常不强调索引
- 重点是判重与去重

你可以把它理解成“名单”或“去重容器”。

适合场景：

- 用户标签
- 权限标识集合
- 已处理过的 ID 集合

例如：

```java
[Java, Go, Java, Python]
```

放进 Set 后会变成：

```java
[Java, Go, Python]
```

### 3. Map：保存 key -> value 的关系

Map 的特点是：

- 通过 key 找 value
- key 一般不能重复
- value 可以重复

你可以把它理解成“字典”或“通讯录”。

适合场景：

- 学号 -> 学生信息
- 用户 ID -> 用户对象
- 配置项 -> 配置值

例如：

```java
{
  "name": "Mo",
  "age": 18
}
```

一句话记忆：

- **List**：有序、可重复
- **Set**：去重、判重
- **Map**：按 key 查 value

## 二、Java 开发里怎么快速选？

如果按最实战的思路压缩成一句话：

- 要“按顺序存多个对象” -> 用 **List**
- 要“保证对象不重复” -> 用 **Set**
- 要“根据一个标识找另一个值” -> 用 **Map**

常见例子：

```java
List<User> users;          // 用户列表
Set<String> permissions;   // 权限标识去重
Map<Long, User> userMap;   // 通过 id 查用户
```

## 三、ArrayList 和 LinkedList 有什么区别？

这是 List 体系里最常见的一组对比。

### 1. 底层结构不同

#### ArrayList

- 底层是**动态数组**
- 内存通常连续
- 可以按下标直接定位元素

你可以把它理解成“一排连续编号的柜子”。

#### LinkedList

- 底层是**双向链表**
- 每个节点除了数据，还记录前后节点引用
- 内存不要求连续

你可以把它理解成“一个个节点手拉手串起来”。

---

### 2. 随机访问能力不同

#### ArrayList：访问快

```java
list.get(index)
```

通常是 **O(1)**。

因为数组知道起始地址后，可以直接算出目标位置。

#### LinkedList：访问慢

```java
list.get(index)
```

通常是 **O(n)**。

因为链表不能直接按下标跳过去，只能顺着节点找。

---

### 3. 插入删除要分场景看

#### ArrayList

- **尾部插入**：平均 O(1)
- **中间插入/删除**：通常 O(n)
- 原因：后面的元素需要整体移动

#### LinkedList

- **头尾插入/删除**：通常 O(1)
- **中间插入/删除**：如果节点已经找到，操作本身很快
- 但“找到节点”往往还是 O(n)

所以真实项目里不能只背“LinkedList 插删快”，因为很多时候：

> **定位成本已经把优势吃掉了。**

---

## 四、真实开发中为什么更常用 ArrayList？

大多数业务场景里，更常用的是 **ArrayList**。

主要原因有三个：

### 1. 读操作远多于频繁中间插删

很多业务代码更常见的是：

- 遍历列表
- 按索引取值
- 顺序读取数据

这些场景都更适合 ArrayList。

### 2. CPU 缓存友好

ArrayList 的连续内存对 CPU 更友好，遍历时实际性能通常更好。

### 3. LinkedList 理论优势在业务里不常完整成立

虽然链表在“已知节点位置”的插删上有优势，但真实业务往往先要查找位置，整体复杂度通常还是不占优。

---

## 五、一个实战选择原则

你可以先这样记：

- 默认优先选 **ArrayList**
- 只有在你明确需要频繁做头尾插删，或者要把它当队列/双端队列使用时，再考虑 **LinkedList**

也就是说：

> **ArrayList 是默认选择，LinkedList 是特定场景选择。**

---

## 六、面试速答版

### List、Set、Map 的区别

List 有序且可重复，适合按顺序保存数据；Set 强调元素唯一性，适合去重和判重；Map 保存 key-value 映射，适合通过 key 快速定位 value。

### ArrayList 和 LinkedList 的区别

ArrayList 底层是动态数组，随机访问快，适合读多写少；LinkedList 底层是双向链表，头尾插删更方便，但随机访问慢。真实开发里通常更常用 ArrayList。
