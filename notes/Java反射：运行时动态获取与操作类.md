# Java反射：运行时动态获取与操作类

> [!abstract] 一句话摘要
> 反射的核心价值，是让程序在编译期不知道具体类的情况下，仍然能在运行时动态获取类信息、创建对象、调用方法和操作字段，因此它广泛应用于 Spring、ORM、序列化等框架底层。

---

## 一、什么是反射？

反射可以直接理解为：

> **程序在运行过程中，可以动态获取一个类的信息，并且操作这个类或对象。**

这里的“类的信息”包括：

- 类名
- 成员变量
- 方法
- 构造器
- 实现的接口
- 继承关系

但如果只记“运行时获取类信息”还不够，反射真正重要的是：

> **它不仅能看，还能动。**

也就是说，反射不仅能知道类长什么样，还能：

- 创建对象
- 调用方法
- 读取和修改字段

---

## 二、反射到底能干什么？

### 1. 动态创建对象

普通代码通常这样创建对象：

```java
User user = new User();
```

反射则可以在运行时根据类名创建：

```java
Class<?> clazz = Class.forName("com.demo.User");
Object obj = clazz.getDeclaredConstructor().newInstance();
```

也就是说：

- 编译时可能不知道具体类
- 运行时再决定创建谁

### 2. 动态调用方法

你可以在运行时找到某个方法并执行它。

这适合：

- 根据配置决定调用哪个类的哪个方法
- 框架自动执行 Controller 或 Service 方法

它不再是手写：

```java
user.login();
```

而是程序自己在运行时查找并调用。

### 3. 动态读取和修改字段

反射还可以获取对象字段，必要时甚至能操作私有字段。

常见用途：

- 数据库结果映射到 Java 对象
- 对象拷贝
- 序列化 / 反序列化

---

## 三、反射最本质的作用是什么？

本质上就是：

> **把原本写死在代码里的类和方法调用，变成运行时动态决定。**

这让程序具备了很强的灵活性。

你可以把它总结成：

- 普通写法解决“确定性”
- 反射解决“动态性”

---

## 四、既然 Java 是强类型语言，为什么还需要反射？

因为强类型写法有一个前提：

> 你在写代码时，就已经知道要操作哪个类、哪个方法、哪个字段。

例如：

```java
User user = new User();
user.login();
```

这很清晰、安全、高效。

但现实中有些场景属于：

> **编译时不知道，运行时才知道。**

这类问题，普通强类型写法就不够灵活。

所以可以这样理解：

- 强类型语言负责编译期约束
- 反射补充运行时动态能力

---

## 五、真实开发里反射常用在哪？

### 1. 框架底层

这是最典型的用途。

例如 Spring 为什么能：

- 自动创建 Bean
- 自动注入依赖
- 自动调用注解标记的方法

原因就是框架不可能提前知道你项目里的所有类，只能在运行时：

- 扫描类
- 读取注解
- 创建对象
- 调用方法

### 2. JDBC / ORM / JSON 框架

例如：

- 查询数据库后把结果封装成对象
- 把 JSON 转成 Java 对象
- 把 Java 对象转成 JSON

这些都需要在运行时分析类结构，并对字段赋值。

### 3. 通用工具开发

例如：

- 对象拷贝器
- 通用校验器
- 注解处理器
- 插件机制

这类工具往往需要在运行时动态分析对象结构。

---

## 六、反射为什么强大？

因为它打破了“必须提前写死”的限制。

普通代码一般要求：

- 我知道类名
- 我知道方法名
- 我知道怎么调用

而反射允许：

- 编译时先不知道
- 运行时再决定

这就是它强大的根源。

---

## 七、反射的代价是什么？

反射并不是业务代码里的默认选择，它有明显代价：

- 性能通常比直接调用低
- 可读性更差
- 过度使用会让程序更复杂
- 可能破坏封装性

所以它更适合出现在：

- 框架
- 通用组件
- 底层工具

而不是普通业务逻辑中滥用。

---

## 八、面试速答版

反射是指程序在运行时动态获取类的信息，并且操作类或对象的机制。它不仅能获取类名、属性、方法、构造器等信息，还可以动态创建对象、调用方法、读取或修改字段。它的核心价值是让程序在编译期不知道具体类的情况下，仍然能够在运行时灵活处理对象，因此广泛用于 Spring、JDBC、ORM、序列化等框架底层。
