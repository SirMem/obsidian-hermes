## 一、目标定义

这 3 天的目标不是把简历上的所有知识都学成“深入专家”，而是做到：

“围绕简历上的核心技术栈与项目内容，建立一套能回答、能解释、能自圆其说的表达框架，使自己在面试中至少能比较稳定地回答 70% 的高频问题。”

这里的“70%”不是指所有细节都懂，而是指：
- 高频基础问题能答
- 项目经历能讲顺
- 简历上的大部分关键词被追问时不会完全卡住
- 即使回答不深，也能保证逻辑清楚、表达真实、方向正确

## 二、先明确：什么叫“能回答 70%”

你 3 天后不需要达到：
- 能把每个底层原理讲到源码级
- 能无死角回答 JVM、Redis、MySQL、MQ 所有细节
- 能把简历里所有技术点都讲到非常深入

你需要达到的是：

### 1. 能讲清楚“我做了什么”
- 两个项目都能顺畅介绍
- 每个亮点都知道它解决什么问题
- 知道为什么这么设计
- 知道自己在里面负责的部分

### 2. 能讲清楚“核心知识主线”
- Java：集合、并发、面向对象、常见设计模式
- MySQL：事务、索引、锁、MVCC、日志
- Redis：为什么快、常见数据结构、缓存作用、持久化
- MQ：为什么要异步、削峰、解耦
- Spring Boot：为什么常用、开发流程大概是什么

### 3. 能对简历上的关键词做基础兜底
例如：
- 责任链模式是什么
- 策略模式是什么
- 规则树是怎么理解的
- DDD 是什么
- Claude Code 怎么帮助开发
- Git / worktree 为什么提高效率

## 三、按“面试命中率”划分优先级

### A 级：必须保住，决定你是否容易被问崩
这部分必须优先拿下。

1. 自我介绍
2. 拼得多系统项目
3. AI Agent 平台项目
4. MySQL
5. Redis
6. Java 基础（集合、面向对象、并发基础）
7. 设计模式（责任链、策略、单例、工厂）

### B 级：高概率追问，但不一定每场都深挖
1. Spring / Spring Boot
2. RabbitMQ / RocketMQ 的作用
3. SQL 优化
4. JVM 基础认知
5. Git 协作流程

### C 级：加分项，有时间再补深
1. DDD
2. AI Agent 架构细节
3. PGVector / RAG / MCP
4. JUC 深层实现
5. 垃圾回收器细节

原则很简单：
先保 A，补 B，最后看精力补 C。

## 四、这 3 天的总体策略

### 核心策略 1：不按“学科”学，按“面试场景”学
错误方式：
- 今天看 6 小时 MySQL 视频
- 明天看 5 小时 Redis 视频
- 后天看 8 小时 Java 八股

这样容易输入很多，但输出很差。

正确方式：
- 每个知识点都整理成“面试回答模板”
- 学完一个主题，就能回答 3~5 个典型问题
- 项目和知识点同步绑定

### 核心策略 2：深度不要平均分配
你不需要每个点都学一样深。

应该这样分层：
- 高频点：要能讲清楚
- 中频点：要知道大概
- 低频深水区：知道关键词和方向即可

### 核心策略 3：优先把“会但讲不出来”变成“能说出来”
你现在最大问题不是纯不会，而是：
- 知道一点
- 看着眼熟
- 但面试时答不成结构

所以复习重点不是“看懂”，而是“能说”。

## 五、3 天具体安排

## Day 1：先把最容易决定面试成败的主干打起来

### 今日目标
- 完成项目表达框架
- 建立 MySQL 主线
- 建立自我介绍与岗位匹配表达

### Task 1：整理 1 分钟自我介绍
输出要求：
- 我是谁
- 我现在的方向是什么
- 我做过哪两个核心项目
- 我擅长什么
- 为什么想来这个岗位

目标：能够自然说出 1 分钟版本。

### Task 2：整理项目 1：拼得多系统
输出要求：
- 项目背景
- 你负责什么
- 用了哪些技术
- 做了哪些亮点
- 每个亮点解决什么问题

重点必须能说清：
- 责任链模板在这里解决什么问题
- 规则树在这里解决什么问题
- Redis 在这里做了什么
- 异步处理为什么能提升体验
- 为什么要用双触达 / 补偿 / 分布式锁

### Task 3：整理项目 2：AI Agent 平台
输出要求：
- 项目做什么
- 你负责什么
- 为什么要做 MCP / RAG / SSE
- 这个系统的价值是什么

重点必须能说清：
- Spring AI 是干什么的
- MCP 接入层解决什么问题
- PGVector / RAG 的作用
- SSE 为什么提升体验

### Task 4：开始 MySQL 第一轮复习
今天只抓主线，不抓太深。

输出要求：
- MySQL 在系统里负责什么
- 为什么要事务
- 索引为什么能提升查询效率
- 锁和 MVCC 是在解决什么问题
- redo log / undo log / binlog 分别是什么

目标：能回答 5 个基础问题，而不是把所有细节都吃透。

### Task 5：岗位适配表达
输出要求：
- 为什么你适合 Java 后端 / 全栈实习
- 为什么你学习快
- 为什么你能配合团队做实现、测试、bug 修复
- Claude Code / Git / worktree 在你这里是怎么转化成效率优势的

### Day 1 完成标准
到今天结束，你至少能做到：
- 自我介绍不结巴
- 两个项目都能讲一个基础版
- MySQL 的主线能说一个基础版

## Day 2：补最容易被追问的中层内容

### 今日目标
- Java 基础高频
- Redis
- 设计模式
- Spring Boot
- Git / 工程协作表达

### Task 1：Java 基础
重点只抓高频：
- List / Set / Map 区别
- ArrayList 和 LinkedList 区别
- HashMap 基本原理
- 面向对象怎么理解
- 反射是干什么的
- 线程和线程池解决什么问题
- CAS / 锁大概在解决什么问题

目标：不追求特别底层，但要能说出主线。

### Task 2：Redis
重点问题：
- Redis 为什么快
- 常见数据结构有哪些
- Redis 在项目里怎么用
- 持久化为什么存在
- Redis 和 MySQL 的分工是什么
- 缓存可能带来什么问题

目标：能围绕“缓存、性能、临时状态”来讲清楚 Redis。

### Task 3：设计模式
围绕你简历里写的几个来准备：
- 单例
- 工厂
- 模板方法
- 责任链
- 策略模式

输出要求：
每个模式至少能答三件事：
- 它是什么
- 它解决什么问题
- 你项目里是怎么用的

### Task 4：Spring / Spring Boot
重点问题：
- Spring Boot 为什么开发效率高
- IoC / AOP 是什么
- Bean 生命周期知道大概即可
- 你项目里为什么用 Spring Boot

### Task 5：Git / worktree / Claude Code
准备表达：
- Git 的日常开发流程
- worktree 为什么适合并行开发
- Claude Code 怎么用于需求拆解、编码、调试、重构

注意：这里不要讲得太“工具炫技”，要讲成“提升开发效率和协作质量”。

### Day 2 完成标准
到今天结束，你至少能做到：
- Java 基础能回答一轮
- Redis 能回答一轮
- 设计模式能和项目绑定起来说
- Spring Boot 不至于一问就空白

## Day 3：面试模拟化，做收口和补漏洞

### 今日目标
- 高频问题口语化
- 补 RabbitMQ / RocketMQ
- 补 SQL 优化
- 做一轮完整模拟
- 统一“不会时怎么答”策略

### Task 1：RabbitMQ / RocketMQ
重点回答：
- 为什么要消息队列
- 什么叫异步、解耦、削峰
- 你项目里为什么用 MQ
- 消息队列会带来什么问题

### Task 2：SQL 优化
准备基础版：
- 为什么 SQL 会慢
- 索引失效的常见原因
- 联合索引和最左前缀
- 如何初步分析一个慢 SQL

### Task 3：项目深挖问题清单
把两个项目各自准备 8~10 个可能追问的问题。
例如：
- 为什么这样设计？
- 为什么不用另一种方案？
- 如果并发量更高怎么办？
- Redis 为什么这样存？
- MQ 为什么适合这里？

### Task 4：模拟面试
至少做一轮完整口述：
- 自我介绍
- 项目 1
- 项目 2
- MySQL
- Redis
- 设计模式
- 为什么适合岗位

重点不是完美，而是把表达从“脑子里知道”变成“嘴上能说”。

### Task 5：准备“不会时怎么答”
你必须准备一套不失分的话术：
- 先承认这个点我掌握得没有那么深入
- 但我目前理解它主要是在解决什么问题
- 在我的项目场景里，我更多是这样使用/理解的
- 如果需要，我可以继续从这个方向展开

这比硬编强太多。

### Day 3 完成标准
到今天结束，你应该能做到：
- 简历上 70% 的内容都能答一个基础版
- 即使被追问，也不至于完全失语
- 项目表达有主线
- 高频八股至少能扛住第一轮

## 六、用“70%达标”来量化检查

面试前一天晚上，你可以按下面表做自测。

### 1. 自我介绍
- 能否 1 分钟讲清楚？
- 能否突出岗位匹配？

### 2. 项目经历
- 拼得多系统是否能讲顺？
- AI Agent 平台是否能讲顺？
- 每个亮点能否解释“为什么这么做”？

### 3. MySQL
- 事务、索引、锁、MVCC、日志能否各讲一句核心定义？

### 4. Redis
- 为什么快？怎么用？和 MySQL 怎么分工？

### 5. Java
- 集合、并发、面向对象能否答基础版？

### 6. 设计模式
- 责任链、策略模式能否结合项目说？

### 7. Spring Boot
- 为什么用？IoC / AOP 是什么？

### 8. MQ
- 为什么要异步 / 削峰 / 解耦？

如果这 8 大块里，你有 5~6 块能稳定输出，另外 2~3 块能答基础版，那就已经非常接近“70%可答”了。

## 七、这 3 天最重要的学习原则

### 原则 1：一切围绕简历，不要发散学习
你现在不是准备长期系统学习，而是准备一场面试。
所以优先级永远是：
“简历写了什么，就先把什么答顺。”

### 原则 2：一切围绕口语表达，不只做阅读输入
每学一个点，都要问自己：
“如果面试官现在问我，我能用 30 秒说出来吗？”

### 原则 3：先保真实，再谈深度
如果某个点你掌握得不深，不要硬装非常熟。
你应该做到的是：
- 基础理解准确
- 项目表达真实
- 深问时知道自己的边界

### 原则 4：你的目标不是满分，是先过线
对于当前你最现实的目标来说：
- 70% 可答
- 不明显失真
- 项目表达顺
- 面试状态稳
这已经足够有竞争力。

## 八、接下来最合理的执行顺序

从现在开始，最适合按这个顺序推进：

1. 先做《自我介绍 + 岗位匹配表达》
2. 再做《拼得多系统项目拆解》
3. 再做《AI Agent 项目拆解》
4. 再做《MySQL 高频面试基础》
5. 再做《Redis 高频面试基础》
6. 再做《Java 基础高频》
7. 最后做《模拟问答》

## 九、一句话总结

这 3 天你的目标不是“变成一个所有知识点都很深的候选人”，而是：

“把简历上已经写出来的内容重新组织成一套能回答、能解释、能自圆其说的表达体系，在面试中稳定拿下 70% 的高频内容。”
